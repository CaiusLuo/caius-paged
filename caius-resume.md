### 1. RedFolio 红笺书 ｜ 高并发内容社区平台

**角色**：Java 后端开发  
**时间**：2025.03 - 2025.05  
**项目类型**：内容社区 / 类小红书 / 微服务 / 高并发读写

#### 项目描述

RedFolio 红笺书是一个基于 Spring Cloud Alibaba 构建的内容社区平台，支持笔记发布、点赞、收藏、关注等核心功能。项目重点解决高并发场景下的数据一致性、缓存同步、异步落库、分布式 ID 生成和系统扩展性问题。

#### 技术栈

Spring Cloud Alibaba、Nacos、Gateway、Feign、MyBatis、MySQL、Redis、Caffeine、RocketMQ、Lua、SaToken / JWT、MinIO、Leaf

#### 模块一：微服务架构与网关鉴权

- 基于 Spring Cloud Alibaba 构建微服务架构，拆分用户、笔记、互动、文件、ID 生成等服务。
- 使用 Nacos 实现服务注册发现与配置管理。
- 使用 Gateway 实现统一路由、统一鉴权、请求过滤与用户身份透传。
- 使用 SaToken / JWT 进行登录认证，全局过滤器解析 Token 后将用户 ID 透传给下游服务。
- 使用 Feign 完成服务间远程调用，降低模块间直接依赖。

#### 模块二：分布式 ID 生成服务

- 整合 Leaf 思路设计分布式 ID 生成服务，对外提供号段模式和雪花算法两种 ID 生成方式。
- 号段模式通过数据库维护业务 tag、max_id、step 等字段，适合订单号、笔记 ID 等高频 ID 申请场景。
- 使用双 Buffer 异步预加载机制，在当前号段剩余比例较低时提前加载下一个号段，减少申请号段时的接口抖动。
- 针对雪花算法时钟回拨问题，设计短时间阻塞等待策略，超过容忍阈值时拒绝生成，避免 ID 重复。
- 通过 Feign 接口向业务服务提供统一 ID 生成能力，减少各业务服务重复实现 ID 逻辑。

#### 模块三：多级缓存体系

- 使用 Redis + Caffeine 构建多级缓存体系，本地缓存承接热点请求，Redis 承接分布式共享缓存。
- 对热点用户信息、笔记详情、互动计数等数据进行缓存，降低数据库访问压力。
- 针对缓存穿透场景，使用空值缓存和布隆过滤器减少非法请求直接打到数据库。
- 针对缓存击穿场景，使用互斥锁或逻辑过期策略保护热点 key。
- 针对缓存雪崩场景，为缓存设置随机过期时间，避免大量 key 同时失效。
- 在笔记更新、删除等写操作后，通过 RocketMQ 广播消息或 Redis Pub/Sub 通知各节点删除本地缓存，保证多实例环境下缓存一致性。

#### 模块四：点赞高并发写链路

- 使用 Redis Bloom Filter 判断用户是否已经点赞，减少重复请求对后端链路的压力。
- 使用 Redis ZSET 存储用户点赞关系和时间戳，支持点赞列表按时间排序。
- 使用 Lua 脚本封装点赞校验、去重、计数更新等多个 Redis 操作，保证操作原子性并减少网络往返。
- 点赞接口优先写 Redis，随后发送 MQ 消息异步落库，降低数据库同步写入压力。
- 使用 RocketMQ 顺序消息，按用户 ID 或笔记 ID 维度进行分区，保障同一对象相关操作的顺序性。
- 消费端结合 RateLimiter 或批量写入策略控制数据库写入速率，避免高峰期打垮数据库。
- 通过异步落库机制，将高频互动操作从同步 DB 写入改造为缓存写入 + MQ 持久化，提升接口响应稳定性。

#### 模块五：对象存储服务

- 支持阿里云 OSS 与 MinIO 两种对象存储实现。
- 通过 Nacos 配置选择不同文件存储策略，实现对象存储策略的动态切换。
- 封装统一文件上传接口，为笔记图片、用户头像等业务提供文件存储能力。

#### 可用简历描述

- 基于 Spring Cloud Alibaba 构建内容社区微服务架构，通过 Gateway 实现统一路由、统一鉴权与用户身份透传。
- 设计分布式 ID 生成服务，支持 Leaf 号段模式与雪花算法，并通过双 Buffer 机制降低号段切换时的性能抖动。
- 使用 Redis + Caffeine 构建多级缓存体系，并通过 RocketMQ 广播或 Redis Pub/Sub 实现本地缓存失效同步。
- 优化点赞高并发写链路，基于 Redis Bloom Filter、ZSET 与 Lua 脚本实现点赞去重、计数更新与原子化操作。
- 使用 RocketMQ 顺序消息与异步落库机制，将高频互动操作从同步数据库写入改造为缓存写入 + MQ 异步持久化。
- 支持 MinIO / OSS 文件存储策略切换，为笔记图片、用户头像等业务提供统一对象存储能力。

### 2. TodayStock 股票数据分析与可视化平台

**角色**：Java 后端开发  
**时间**：2024.11 - 2025.01  
**项目类型**：数据采集 / 股票分析 / 定时任务 / MQ / 分库分表

#### 项目描述

TodayStock 是一个基于 Spring Boot 构建的股票数据采集、清洗、入库与可视化分析平台，支持股票行情数据定时采集、异步处理、分表存储、热点缓存与图表展示。

#### 技术栈

Spring Boot、RabbitMQ、XXL-Job、MyBatis、MySQL、Sharding-JDBC、Redis、Caffeine、Spring Security、JWT、Vue3、ECharts

#### 模块一：定时任务调度

- 使用 XXL-Job 搭建统一任务调度中心。
- 将股票行情采集、清洗、入库等流程拆分为独立定时任务。
- 支持任务执行日志查看、失败重试与调度状态管理。
- 避免传统 Spring 定时任务在多实例部署下重复执行的问题。

#### 模块二：异步采集与削峰

- 使用 RabbitMQ 将行情采集、数据清洗、数据入库等流程进行异步解耦。
- 采集任务只负责拉取原始行情数据并发送消息，后续由消费者异步处理。
- 消费端根据业务能力控制消费速率，降低数据库瞬时写入压力。
- 通过 MQ 提升链路可扩展性，便于后续增加数据清洗、告警、分析等消费者。

#### 模块三：分库分表与查询优化

- 使用 Sharding-JDBC 按时间维度和股票代码进行水平分表。
- 将大表数据拆分为多个物理表，降低单表数据量。
- 针对常见查询条件设计索引，例如股票代码、交易日期、时间范围等。
- 优化历史行情查询、按股票代码查询、按时间范围查询等场景。

#### 模块四：多级缓存与接口优化

- 使用 Redis 缓存热点股票行情、最新行情和常用查询结果。
- 使用 Caffeine 作为本地缓存，减少短时间内重复请求对 Redis 和数据库的压力。
- 针对热点数据设置合理过期时间，避免缓存雪崩。
- 查询链路优先读取本地缓存，其次读取 Redis，最后访问数据库。

#### 模块五：权限与可视化

- 使用 Spring Security + JWT 实现用户登录认证与接口鉴权。
- 前端使用 ECharts 展示股票行情趋势、涨跌幅和分析结果。
- 后端提供行情查询、历史数据查询、图表数据聚合等接口。

#### 可用简历描述

- 基于 Spring Boot 构建股票数据采集与分析平台，支持行情数据定时采集、清洗、入库与可视化展示。
- 使用 XXL-Job 实现股票数据采集与清洗任务调度，支持任务执行状态监控与失败重试。
- 使用 RabbitMQ 对采集、清洗、入库流程进行异步解耦，降低主流程阻塞与数据库写入峰值压力。
- 基于 Sharding-JDBC 按时间与股票代码进行水平分表，优化大数据量行情数据的写入与查询性能。
- 使用 Redis + Caffeine 构建多级缓存体系，缓存热点行情数据和查询结果，降低数据库访问压力。
- 集成 Spring Security + JWT 实现接口鉴权，结合 ECharts 完成行情数据可视化展示。
